# Использование хелперов для шаблонов форм

Play предоставляет различные хелперы чтобы помочь вам отрендерить поля формы в HTML шаблонах.

## Создание тэга `<form>` 

Первый хелпер создает тэг `<form>`. Это достаточно простой хелпер, который автоматически устанавливает параметры `action` и `method` тэга `<form>` в соответствии с правилами маршрутизации для данного контролера:       

@[form](code/javaguide/forms/helpers.scala.html)

Вы также можете передать дополнительный набор параметров, который будет добавлен к сгенерированному HTML:

@[form-with-id](code/javaguide/forms/helpers.scala.html)

## Рендеринг элемента `<input>`

Есть различные хелперы для input-ов в пакете `views.html.helper`. Вы скармливаете им поле формы и они отображают соответствующую компоненту формы HTML с установленным значением, ограничениями и ошибками:   

@[full-form](code/javaguide/forms/fullform.scala.html)

Что касается хелпера `form`, вы можете определить дополнительный набор параметров, который будет добавлен к сгенерированному HTML:

@[extra-params](code/javaguide/forms/helpers.scala.html)

> **Замечание:** Все дополнительные параметры будут добавлены в сгенерированный HTML HTML, за исключением тех, чьи имена начинуются с символа `_`. Аргументы, начинающиеся с подчеркивания являются зарезеривированными для аргумента конструктора поля (который мы увидим позже).

## Конструкторы поля

Отрендеренное поле не только содержит тэг `<input>`, но также может нуждаться в `<label>` и в пачке других тэгов, используемых вашим CSS фреймворком для декорирования поля.

Все хелперы полей ввода принимают неявный `FieldConstructor`, который управляет этой частью. Конструктор по-умолчанию (используется, если ни одного другого конструктора полей нет в области видимости), генерирует HTML следующего вида: 
     
```
<dl class="error" id="email_field">
    <dt><label for="email">Email:</label></dt>
    <dd><input type="text" name="email" id="email" value=""></dd>
    <dd class="error">Обязательное поле!</dd>
    <dd class="error">Другая ошибка</dd>
    <dd class="info">Required</dd>
    <dd class="info">Другое ограничение</dd>
</dl>
```

Этот конструктор поля поддерживает дополнительные опции, которые вы можете передать в аргументах хелпера input:

```
'_label -> "Подпись для поля"
'_id -> "idForTheTopDlElement"
'_help -> "Подсказка для поля"
'_showConstraints -> false
'_error -> "Форсировать ошибку"
'_showErrors -> false
```

### Написание собственного конструктора поля

Часто бывает, что вам нужно написать собственный конструктор поля. Начните с написания подобного шаблона: 

@[template](code/javaguide/forms/myFieldConstructorTemplate.scala.html)

Сохраните его во `views/` и дайте имя `myFieldConstructorTemplate.scala.html`

> **Замечание:** Это всего лишь пример. Вы можете усложнить его настолько, насколько это неоходимо. Вы также имеете доступ к оригинальному полю через `@elements.field`.

Теперь создайте `FieldConstructor` где-то, используя:

@[field](code/javaguide/forms/withFieldConstructor.scala.html)
