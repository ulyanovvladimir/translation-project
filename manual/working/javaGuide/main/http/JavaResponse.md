# Манипулирование ответом

## Изменение Content-Type по умолчанию

Тип контента результата автоматически выводтся из Java-значения, которое вы определяете в качестве тела.

Например:

@[text-content-type](code/javaguide/http/JavaResponse.java)

Автоматически установит `Content-Type` заголовок в `text/plain`, в то время как:

@[json-content-type](code/javaguide/http/JavaResponse.java)

Установит заголовок `Content-Type` в `application/json`.

Это очень удобно, но иногда вы возможно захотите изменить это. Просто используйте метод `as(newContentType)` на результате для создание нового похожего результата с другим заголовком `Content-Type`:

@[custom-content-type](code/javaguide/http/JavaResponse.java)

## Установка загооловков HTTP ответов

@[response-headers](code/javaguide/http/JavaResponse.java)

Обратите внимание, что установка заголовков HTTP автоматически отменит любое предыдущее значение.

## Установка и отмена cookies

Cookies  - всего лишь специальная форма заголовков HTTP, Play предоставляет множество хелперов для упрощения.

Вы можете просто добавить Cookie в HTTP ответ:

@[set-cookie](code/javaguide/http/JavaResponse.java)

Если вам нужно больше информации, включая путь, домен, срок годности, безопасное ли соединение, и должен ли быть установлен только флаг HTTP, вы можете сделать это с перекрыващими методами:

@[detailed-set-cookie](code/javaguide/http/JavaResponse.java)

Чтобы отменить Cookie ранее сохраненную в веб браузере:

@[discard-cookie](code/javaguide/http/JavaResponse.java)

Если вы установите путь или домен, когда устанавливаете cookie, убедитесь, что вы установливаете тот же путь или домен когда отменяете cookie, поскольку браузер отменит его только при совпадении имени, пути и домена.

## Определение кодировки символов для текстовых результатов

Для текстовых HTTP ответах очень важно корректно управлять  кодировкой символов. Play управляет этим за вас, используя по умолчанию `utf-8`.

Кодировка используется и для конвертирование текстового ответа в соответствующие байты для отправки по сети и для добавления подходящего `;charset=xxx` расширения в заголовок `Content-Type`.

Кодировка может быть определена, когда вы генерируете значение `Result`:

@[charset](code/javaguide/http/JavaResponse.java)
