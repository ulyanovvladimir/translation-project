# Инсталлируем Play

## Требования

Вам необходимо иметь JDK 1.8 (или новее) установленный на вашей машине (см. [Этапы общей инсталляции](#JDK-installation)).

## Быстрый старт

1. **Скачать** последнюю версию [Typesafe Activator](https://typesafe.com/get-started).
2. **Извлечь** архив в то местоположение, где у вас есть доступ на запись.
3. **Изменить** текущую директорию с помощью команды `cd activator*` (либо с помощью файл-менеджера)
4. **Запустить** активатор с помощью команды `activator ui` (либо с помощью файл-менеджера)
5. **Откройте** его по адресу [http://localhost:8888](http://localhost:8888)

Вы сразу же получите документацию и список шаблонов приложений, которые вы собираетесь получить. Для простого старта используйте пример **play-java**.


### Командная строка

Чтобы использовать play из любого местоположения в вашей файловой системе, добавьте папку
**activator** в вашу переменную path (см. [Этапы общей инсталляции](#Add-Executables-to-Path)).

Создание `my-first-app` на основе шаблона `play-java` проходит так же просто как:

```bash
activator new my-first-app play-java
cd my-first-app
activator run
```

[http://localhost:9000](http://localhost:9000) - здесь доступ к вашему приложению.

Теперь вы готовы работать с Play!

## Этапы общей инсталляции

Вам возможно потребуется иметь дело с этими подзадачами для того чтобы установить Play! на вашу систему. 

### Установка JDK 

Убедитесь, что у вас установлена JDK (Java Development Kit) Версиии 1.8 или выше. Просто используйте эти команды для верификации:

```bash
java -version
javac -version
```

Если у вас нет JDK, вам необходимо его установить:

1. **MacOS**, Java встроена, но вам, возможно, потребуется [обновить до последней версии](http://www.oracle.com/technetwork/java/javase/downloads/index.html)
2. **Linux**, используйте либо последний Oracle JDK или OpenJDK (не используйте не gcj). 
3. **Windows** просто скачайте и становите [последний JDK](http://www.oracle.com/technetwork/java/javase/downloads/index.html).


### Добавьте путь до JDK в PATH

Для удобства, вам следует добавить директорию с установленным Активатором в вашу переменную среды окружения `PATH`.

На **Unix**, используйте `export PATH=/path/to/activator:$PATH`

На **Windows**, добавьте `;C:\path\to\activator` в вашу переменную среды окружения`PATH`. Не используйте путь, содержащий пробелы.

### Разрешения на уровне файлов

#### Unix

В процессе работы `activator` пишет некоторые файлы в директории, куда установлен, поэтому не устанавливайте в `/opt`, `/usr/local` или в любое другое место, где вам требуется дополнительные разрешения на запись.

Убедитесь, что скрипт `activator` запускается. Если нет выполните `chmod u+x /path/to/activator`.

### Установка Proxy

Если вы за прокси-сервером, убедитесь, что прописали его с помощью `set HTTP_PROXY=http://<host>:<port>` в Windows или `export  HTTP_PROXY=http://<host>:<port>` в UNIX.
